## Scope confirmation

- [ ] Purpose and user role are clear.
- [ ] Required workflow states are included.
- [ ] Responsive viewports are identified.
- [ ] Dependencies and integrations are identified.
- [ ] Synthetic data only is used.
- [ ] Known exclusions and future capabilities are documented.

## Design-system confirmation

- [ ] Tokens match `design/tokens.json`.
- [ ] Component families match `design/components.json`.
- [ ] No unnecessary copied/detached components remain.
- [ ] Component variants cover default, interaction, loading, disabled and error states where relevant.
- [ ] Layer and component naming is consistent.
- [ ] Developer annotations are present.

## Accessibility confirmation

- [ ] Keyboard order reviewed.
- [ ] Focus states visible.
- [ ] Contrast reviewed.
- [ ] Touch targets reviewed.
- [ ] Text scaling and content expansion reviewed.
- [ ] Screen-reader labels/announcements annotated.
- [ ] Charts, maps and tables have alternatives.
- [ ] Low-bandwidth/offline states reviewed.
- [ ] Language and SASL/interpreter patterns reviewed.

## Clinical and security confirmation

- [ ] Clinical facts, patient-reported content and suggestions are distinguished.
- [ ] Decision support is advisory and human-reviewed.
- [ ] Stale, missing and incomplete data states exist.
- [ ] Permissions and role/facility/patient context are annotated.
- [ ] Audit-event expectations are documented.
- [ ] Consent and privacy requirements are represented.
- [ ] Privileged actions require confirmation or reauthentication where appropriate.
- [ ] Integration status is accurately labelled.
- [ ] No recording workflow is shown as default without approved policy.
