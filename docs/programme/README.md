# Secor HealthConnect Programme Governance

This directory contains the controlled mobilisation, governance and discovery baseline for Milestone 01.

## Controlled documents

| Document | Purpose |
|---|---|
| [Programme charter and benefits case](./PROGRAMME_CHARTER.md) | Defines mandate, outcomes, scope, principles and expected public-value benefits |
| [Stakeholder register and RACI](./STAKEHOLDERS_AND_RACI.md) | Defines stakeholder groups, responsibilities, decision rights and escalation ownership |
| [Governance operating model](./GOVERNANCE_OPERATING_MODEL.md) | Defines forums, cadence, quorum, decisions, artefacts and escalation paths |
| [RAID register](./RAID_REGISTER.md) | Records initial risks, assumptions, issues and dependencies |
| [Reporting calendar and SLAs](./REPORTING_CALENDAR_AND_SLAS.md) | Defines weekly, sprint, monthly, quarterly and incident-reporting obligations |
| [Discovery findings](./DISCOVERY_FINDINGS.md) | Records the initial clinical, facility, connectivity, integration and delivery findings |
| [Architecture, security and data-classification baseline](./ARCHITECTURE_SECURITY_DATA_BASELINE.md) | Establishes the approved Azure-native technical and control baseline |
| [Milestone 01 completion and acceptance report](./MILESTONE_01_COMPLETION_REPORT.md) | Maps milestone criteria to repository evidence and records completion status |

## Source of truth

GitHub Issues, Milestones and the organisation-level GitHub Project are the engineering delivery source of truth. Azure Boards may receive synchronised or exported work items when required by a Microsoft programme-management process, but it must not be maintained as an independent competing backlog.

## Control statement

This baseline applies to the repository mobilisation and demonstration programme. It does not constitute Department of Health production approval, clinical accreditation, a data-processing authorisation or permission to use real patient information. Those approvals remain explicit pilot and production gates.
